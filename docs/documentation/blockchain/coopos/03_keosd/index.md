!!! note "На русском"
    **keosd** — демон кошелька для **cleos**. Ниже — полный текст раздела.

## Введение

`keosd` — фоновый сервис управления ключами: хранение закрытых ключей и подпись сообщений. Обеспечивает защищённое хранилище ключей с шифрованием «на диске» в файле кошелька. `keosd` задаёт защищённую среду для подписи транзакций, созданных `cleos` или сторонней библиотекой.

## Установка

`keosd` входит в [репозиторий COOPOS](https://github.com/coopenomics/coopos). Установка — в разделе [Установка ПО COOPOS](../00_install/index.md).

## Работа

Когда кошелёк разблокирован паролем, `cleos` может запросить у `keosd` подпись транзакции соответствующими закрытыми ключами. `keosd` также поддерживает аппаратные кошельки (например, Secure Enclave и YubiHSM).

!!! note "Аудитория"
    `keosd` предназначен в основном для разработчиков COOPOS.
